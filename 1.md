# DigitalPlat 域名自动续期（Node.js 版）

基于 katabump 项目的反检测方案，使用 Node.js + playwright-extra + puppeteer-extra-plugin-stealth 实现。

## 核心方案

- **独立启动 Chrome + CDP 连接**：避免 Playwright 直接启动留下的自动化特征
- **puppeteer-extra-plugin-stealth**：完整的浏览器指纹伪装
- **Turnstile 绕过**：Hook Shadow DOM 获取坐标 + CDP 真实鼠标点击
- **ALTCHA 绕过**：Shadow DOM 定位 + CDP 点击触发 PoW 计算
- **全程浏览器操作**：登录、获取域名、续期全部在浏览器中完成，携带完整指纹

## 环境变量

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `DOMAIN_ACCOUNT` | 是 | 账号配置，格式：`邮箱:密码,邮箱2:密码2` |
| `TELEGRAM_BOT_TOKEN` | 否 | Telegram Bot Token，用于发送通知 |
| `TELEGRAM_CHAT_ID` | 否 | Telegram Chat ID |
| `HTTP_PROXY` | 否 | HTTP 代理，格式：`http://user:pass@host:port` |
| `CHROME_PATH` | 否 | Chrome 路径，默认 `/usr/bin/google-chrome` |

## 本地运行

```bash
# 安装依赖
npm install

# 运行
DOMAIN_ACCOUNT="邮箱:密码" node do_renew.js
```

## GitHub Actions 部署

1. 将本项目推送到 GitHub
2. 在仓库 Settings → Secrets and variables → Actions 中添加以下 Secrets：
   - `DOMAIN_ACCOUNT`：账号配置
   - `TELEGRAM_BOT_TOKEN`（可选）
   - `TELEGRAM_CHAT_ID`（可选）
   - `HTTP_PROXY`（可选）
3. Workflow 会每天自动运行，也可以手动触发
